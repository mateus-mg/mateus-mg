# Portfolio Site Design Spec

**Date:** 2025-04-26  
**Topic:** mateus-mg-portfolio-site  
**Status:** Approved

---

## Goal

Build a lightweight, maintainable static portfolio site for GitHub Pages that displays the user's profile, tech stack, education, and dynamically loaded GitHub repositories.

---

## Architecture

Single-page static site (HTML/CSS/JS) hosted on GitHub Pages. A GitHub Actions workflow fetches public repositories via the GitHub API daily and on every push, pre-rendering them into a local `data/repos.json` file. The frontend loads this JSON at runtime with vanilla JavaScript, avoiding live API dependency and rate limits.

---

## Tech Stack

- **Hosting:** GitHub Pages (from `main` branch root)
- **Frontend:** HTML5, CSS3 (Flexbox/Grid), JavaScript (ES6+, no frameworks)
- **CI/CD:** GitHub Actions (workflow to fetch repos and commit `data/repos.json`)
- **Data Source:** GitHub REST API v3

## Data Flow

```
User pushes to main
    |
    v
GitHub Actions workflow triggers
    |
    v
Fetch repos from api.github.com/users/mateus-mg/repos
    |
    v
Filter and transform to JSON array
    |
    v
Write to data/repos.json
    |
    v
Commit and push back to main
    |
    v
GitHub Pages rebuilds
    |
    v
Browser loads index.html + data/repos.json
    |
    v
JS renders project cards dynamically
```

---

## GitHub Actions Workflow

**Trigger:**
- `push` to `main` branch
- `schedule` cron: daily at 00:00 UTC
- `workflow_dispatch` (manual)

**Steps:**
1. Checkout repository
2. Setup Python
3. Run script to fetch repos from GitHub API
4. Filter out forks (optional) and select fields: name, description, html_url, language, stargazers_count, topics
5. Save as `data/repos.json`
6. Commit and push if changed

**API Rate Limit:** For public repos, 60 requests/hour unauthenticated. Since the workflow runs server-side, we can use the built-in `GITHUB_TOKEN` for higher limits (1000/hour).

## Performance

- Zero external dependencies (no CDN fonts, no JS frameworks)
- Single HTML file, single CSS file, single JS file
- JSON data file loaded asynchronously
- Target: <100KB total initial load, <50ms render
